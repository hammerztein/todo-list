# The Odin Project - Todo List Project

This is a todo list application built for [The Odin Project](https://www.theodinproject.com/) JavaScript course, based on the OOP Principles lesson.

## Assignment

Create a todo list app using Webpack and a modular JavaScript architecture following OOP, SRP, and separation of concerns principles.

## Links

[Live app preview](https://hammerztein.github.io/todo-list/)

## Things I Used

- Webpack module bundler
- ESM modules
- JavaScript classes
- Separation of concerns:
  - Application logic (project/todo operations)
  - UI logic (DOM rendering and events)
  - Class models (Project and Todo classes)
  - Persistence (localStorage handling)
- Local storage
- Dialog modals
- Event delegation

## What I Learned

- How to split application structure based on responsibility
- How to hydrate class instances from JSON for local storage

## Features

- Create, edit, and delete projects
- Add, edit, and delete todos inside each project
- Separate todo lists for done and not done items
- Todo status and optional due date
- Persistent data in browser local storage
